Detailed Description

Base class for Duin applications.

Application manages the engine's main loop, window creation, and subsystem coordination. User applications inherit from this class and override virtual methods to implement game logic.

Lifecycle order:

  1. Initialize() - one-time setup
  2. Ready() - called after all systems initialized
  3. Main loop:
    • OnEvent() - input events
    • Update() - per-frame logic (variable timestep)
    • PhysicsUpdate() - fixed timestep (60 Hz default)
    • Draw() - 3D rendering
    • DrawUI() - ImGui rendering
  4. Exit() - cleanup
class MyGame : public duin::Application {
public:
void Initialize() override {
SetWindowName("My Game");
SetWindowStartupSize(1280, 720);
}
void Update(double delta) override {
// Game logic
}
};
duin::Application* duin::CreateApplication() {
return new MyGame();
}
