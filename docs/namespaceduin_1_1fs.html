Detailed Description

Filesystem utilities for file and directory operations.

This namespace provides cross-platform filesystem operations built on top of SDL3's filesystem API. All paths use Unix-style forward slashes (/) as separators, regardless of the underlying platform.

Error Handling:

  • Functions returning std::string return INVALID_PATH on error
  • Functions returning bool return false on error, true on success
  • Use IsPathInvalid() to check if a path string is valid
See also
https://wiki.libsdl.org/SDL3/CategoryFilesystem

◆ EnsureUnixPath()

std::string duin::fs::EnsureUnixPath ( const std::string &  path)

Convert Windows-style paths to Unix-style paths.

Replaces all backslashes () with forward slashes (/) to normalize path separators to Unix-style. This ensures consistent path handling across all platforms.

Parameters
pathThe path to convert (may contain backslashes)
Returns
A new string with all backslashes replaced by forward slashes
Note
This function operates on a copy of the input string (pass-by-value)
If the path already uses Unix-style separators, it is returned unchanged
Mixed separators (both \ and /) are normalized to all forward slashes
This is automatically applied to paths returned by SDL3 filesystem functions
Works with Windows drive letters: "C:\\path" becomes "C:/path"
Works with UNC paths: "\\\\server\\share" becomes "//server/share"

Example:

std::string winPath = "C:\\Users\\Name\\Documents\\file.txt";
std::string unixPath = duin::fs::EnsureUnixPath(winPath);
// unixPath = "C:/Users/Name/Documents/file.txt"
std::string mixed = "path\\to/mixed\\separators";
std::string normalized = duin::fs::EnsureUnixPath(mixed);
// normalized = "path/to/mixed/separators"
std::string alreadyUnix = "path/to/file.txt";
std::string unchanged = duin::fs::EnsureUnixPath(alreadyUnix);
// unchanged = "path/to/file.txt"

◆ MapVirtualToSystemPath()

std::string duin::fs::MapVirtualToSystemPath ( const std::string &  path)

Map a virtual path to an absolute system path.

Converts Godot-style virtual paths (e.g., "bin://", "app://") to absolute system paths. This provides a convenient way to reference application directories without hardcoding absolute paths.

Supported virtual path prefixes:

  • bin:// - Maps to the application's base directory (executable location)
  • app:// - Maps to the application's user data directory (requires SetPrefPath)
  • usr:// - Maps to the user's folder (NOT YET IMPLEMENTED - returns INVALID_PATH)
Parameters
pathVirtual path starting with a recognized prefix (e.g., "bin://config.ini")
Returns
Absolute system path, or INVALID_PATH if the prefix is unrecognized or path is too short
Note
Path must be at least 6 characters ("bin://x")
For "app://", you must call SetPrefPath() first to set org/app names
If SetPrefPath() wasn't called, "app://" paths will fail (GetPrefPath returns INVALID_PATH)
Returns INVALID_PATH for "usr://" (not yet implemented)
Returns INVALID_PATH if path is shorter than minimum length
The returned path uses Unix-style separators (forward slashes)
If the virtual prefix doesn't match any known prefix, returns INVALID_PATH
See also
SetPrefPath()
GetBasePath()
GetPrefPath()

Example:

// First, set preferences for app:// paths
duin::fs::SetPrefPath("MyCompany", "MyGame");
// Map bin:// to executable directory
std::string configPath = duin::fs::MapVirtualToSystemPath("bin://config.ini");
// configPath = "/path/to/executable/config.ini"
// Map app:// to user data directory
std::string savePath = duin::fs::MapVirtualToSystemPath("app://saves/game1.sav");
// savePath = "/home/user/.local/share/MyCompany/MyGame/saves/game1.sav" (Linux)
// Invalid: path too short
std::string invalid1 = duin::fs::MapVirtualToSystemPath("bin:");
// invalid1 = INVALID_PATH
// Invalid: unrecognized prefix
std::string invalid2 = duin::fs::MapVirtualToSystemPath("xyz://file.txt");
// invalid2 = INVALID_PATH
// Not yet implemented
std::string notImpl = duin::fs::MapVirtualToSystemPath("usr://documents/file.txt");
// notImpl = INVALID_PATH

Set global organization and application names for virtual path resolution.

Stores organization and application names in global variables that are used by MapVirtualToSystemPath() when resolving "app://" virtual paths. This allows you to set these values once at application startup and use virtual paths throughout your application without repeatedly specifying org/app names.

Parameters
orgOrganization name (e.g., "MyCompany")
appApplication name (e.g., "MyGame")
Returns
Always returns true (currently cannot fail)
Note
This is a global setting that affects MapVirtualToSystemPath()
Should be called once during application initialization
Empty strings are allowed but may cause INVALID_PATH from MapVirtualToSystemPath()
Thread safety: Not thread-safe - set before multi-threaded operation begins
See also
GetPrefPath()
MapVirtualToSystemPath()

Example:

// At application startup
duin::fs::SetPrefPath("MyCompany", "MyGame");
// Later, use virtual paths without specifying org/app again

// sysPath = "/home/user/.local/share/MyCompany/MyGame/settings.json" (on Linux)
